About the project

FastMap.ai

FastMap.ai is a unique application of deep learning that enables the user to automate the laborious task of mapping. A task that traditionally takes a trained technician hours of conentrated effort can now be completed in a matter of seconds with a substantially lower risk for human error. Reducing the human burden of mapping allows organizations to more easily maintain timely, wide scaled, and precise geographic information thus reducing uncertainty, error, cost, and ultimately providing greater confidence in their geographic-based decisions.

Methods

FastMap.ai was developed over a 3 week concentrated effort and can be described in three distince phases

  • Data Preparation
  • Training the Model
  • Deployment

Data aquisition

The dataset consists of neighborhood-scale aerial imagery collected by various means of drone and aircraft photography. 72 high resolution zone images covered 12 various African cities. Each zone was supported with labels in the form of human-drawn building footprints (vectorized polygons). The dataset was compiled into a convenient resource for a DrivenData competition and is publicly available.

Image and label tiling

In order to efficiently train an image segmentation model, both the image data and the polygon labels needed to be cut into small tiles. The tiles needed to be organized such that the label tile and image tile would correspond to the same geographic space.

Train/validation split

The sets of tiles making up each zone were split into standard 80% training/20% validation sets through random assignment. This strategy seemed like the most dependable method to avoid overfitting due to geographic interdependencies.

Burning the polygon labels

Image segmentation models need the labels to be in the same format as the image data (tensors/arrays). In this case the labels were provided in the form of vector data, which is essentially geometric coordinates of the shapes. This meant that each polygon needed to be be "burned" into a pixel based mask while maintaining its spatial relation. Initially the burned masks were 3 channels: footprint, parameter, and contact. In order to alieve certain deployment issues the masks were instead burned into a single channel representing the building footprints alone.

Fast.ai

This project utilized the accessible and simple Fast.ai python framework. Fast.ai provides an easy API for accessing PyTorch neural network backends.

Transfer learning

Transfer learning allows quick and easy cutting edge results by harnessing generalized pretrained models on novel and uniqe tasks. In this project, Resnet 34 and Resnet 50 models were used. The most recent model, which has yielded the best results was a Resnet 34 with a Unet architecture, trained on progressive image sizes (128, 256) with an unfrozen head.

Dice coefficent

This project relied on the Dice Coefficent as a performance metric. This is a frequently used metric for binary segmentation tasks and is essentially twice the number of pixesl of ovelap, divided by the total numbe of pixels. The highest Dice Coefficent achieved was .898

Train and validation loss

The model also maintained a balance between bias and variance, with the highest performing model achieving .08 validation loss.
